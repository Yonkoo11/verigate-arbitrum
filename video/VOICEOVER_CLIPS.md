# Verigate Demo Video — Voiceover Script

Target: 70-80 seconds. 7 clips.

## Clip 01: hook
**Frame:** Hero split — denied left, approved right, gate line center
**Voice:**
Three billion dollars in tokenized real-world assets on BNB Chain. And right now, there's no native way to enforce who can hold them.

## Clip 02: problem
**Frame:** BSCScan showing the deployed factory contract (verified)
**Voice:**
Existing projects use simple address whitelisting. That works for minting. But the moment those tokens hit a DEX or a lending protocol, the whitelist can't follow.

## Clip 03: solution
**Frame:** Dashboard connected — token overview + compliance status panels
**Voice:**
Verigate checks every transfer against on-chain attestations from BAS, the BNB Attestation Service. Three compliance modules run on every transfer. Country restrictions. Accredited investor verification. Holder cap enforcement.

## Clip 04: blocked
**Frame:** Transfer form showing red "Blocked" compliance result
**Voice:**
Try to transfer without a valid attestation, and the transaction reverts. The compliance engine tells you exactly why. Which module blocked it. What's missing.

## Clip 05: approved
**Frame:** Transfer form showing green "Transfer is compliant" result
**Voice:**
Add the attestation, try again, and the transfer goes through. That's the core loop. Verify, then transfer.

## Clip 06: issuer
**Frame:** Issuer admin panel — blocked countries KP/IR/SY, module list, mint controls
**Voice:**
Token issuers get a full admin panel. Mint tokens. Map attestations to wallets. Freeze addresses. Configure country restrictions. All on-chain. All modular.

## Clip 07: close
**Frame:** Hero split again (full page, showing the gate metaphor)
**Voice:**
Verigate. Built on BAS. Native to BNB Chain. Open source. Compliance infrastructure for the next wave of tokenized assets.
